Xcode Starter Guide (1) - Learn How To Use Xcode

This will be a Beginner to Xcode guide, which will walk you through all the basic usage of Xcode. I will be writing down all the points and areas you need to know before you type any code to make an iOS app.

Using a unfamiliar tools can be scary. You definitely don't want to be exposed to an environment with full of unknowns before you do anything. This guide will be especially useful to read for those who are going to start writing their first app.

The source code of this tutorial is available on Github.

Install Xcode

Xcode is the start of everything. For starter, the best way to install Xcode is installing from the App Store app in your Mac. Just hit download and install (might take up some time).

  • Xcode from App Store will always be the latest and non beta
  • Major versions of Xcode will be aligned with major versions of iOS
  • e.g. Xcode 9.x is relative to iOS 11.x, Xcode 8.x is relative to iOS 10.x
  • When a new beta iOS version comes out, there will also be a new beta Xcode available at the Apple Developer website
  • Your app can be run in higher iOS version when the app is built in a lower version of Xcode (upward compatible)
  • If you want to develop latest features, you must keep Xcode up to date

You can have multiple Xcode versions installed on the same Mac, but be careful of the naming conflict. It might cause unexpected troubles.

Other versions of Xcode (if available): https://developer.apple.com/download/

You also need to install additional required components.

Open Xcode

When you first open, you should see the below pop up.

Get started with a playground : Playground is a place that you play around with code. You write code in playground and it will be auto compiled to show your result. It is powerful and come in handy when you are developing in a main project, you open a playground to test some code without writing into the main project.

Create a new Xcode project : You will be mostly doing this.

Clone an existing project : Get a copy of a project from a remote source.

Click Create a new Xcode project.

Create a new Xcode project

Select Single View App and click Next. There are also other templates and options for you to start with, but a single view app is the easiest one to let us walk through Xcode and the others are more advanced topics.

Then you will see you need to input information for your project. Just input the information:

Product Name: Input our project name. It will be your app name at first too.

Team: Select None. It you have an apple developer account logged in Xcode. Your personal team or company team will appear here.

Organization Name: Your team/group/company name.

Organization Identifier and Bundle Identifier: You will need a Bundle Identifier(Bundle Id) to uniquely identify your app in your team. The convention will be com.company.appname. This is the most important one because it is related to your provisioning profiles and certificate things.

Language: Swift

Use Core Data, Include Unit Tests, Include UI Tests: optional

No worries if you are scared of inputting anything because anything can be changed afterwards upon your needs. Just quickily get it started, in addition that this is only a demo project.

Xcode Interface

There are multiple panels when you open Xcode. First expand all the panel first by toggling the control at the top-right and also the one in the bottom.

Project Panel & Main Panel

Your project structure goes here. The Main Panel will show what you click in the project panel. So now it is selecting the starter.xcproject which contains the whole project settings including Bundle Id, Signing, Targets, Capabilities and things that, as important as your code, are involved in building the app to the App Store.

Project Structure & File Detail

Under the starter.xcproject, there are folders and files in starter folder. This is your main project stucture. When you select a file, the main panel will change accordingly. You can also the file detail on the right like what is its target, where is it located etc.

There is also a starterTests, it exists because I checked Include Unit Tests in the beginning. Just bear in mind that it is completely optional.

The remaining is the Product folder. It shows the app you build in Xcode. 99.99% of time you will not touch or edit the app you build in this folder. Just leave the folder is ok.

Variable & Runtime Console

You also see these two debug console. If you had breakpoints in your code and run, you will see the values of your variables in runtime on the left. Also the common logging and debug information will be shown on the right. I will show some code in the next story to illustrate the use of these.

Helper Panel

You can drag some handy things from this panel when you are writing code or constructing your views. You can drag template code blocks and UI elements like UIButton and UIScrollView etc..

Summary

So above is a basic walk through with the Xcode interface. Sometimes you will not be needed all panels are open to do work. Just toggle it for more extra space as you want. I will continue explaining and go into more coding, app entry point and simulator stuff next story. Hope this tutorial find you useful. Happy coding.
