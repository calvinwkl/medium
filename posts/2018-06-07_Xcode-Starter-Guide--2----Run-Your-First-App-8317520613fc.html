So wait, where is the white screenÂ from?

Long answer short first, it will be your ViewController.swift in your project panel. Because we selected a single view application template when we created this project, Xcode automatically linked the ViewController.swift for us.

What is a View Controller? or you say UIViewController? Storyboard?

Think of a View Controller (vc) is a thing that control a view and present it to the App.

A unit of screen.

When we say this vc push to another vc, or that vc transit to anther vc. That means screen is changing because the vc at the top changes (we now donâ€™t need to know how they are changed right now). A UIViewController is the class to make the vcs. Now the ViewController.swift is actually:

  • a instance of UIViewController (class)
  • named ViewController (just a pre-defined name)
  • a View Controller (vc) (don't be confused with the name)

And a Storyboard is a place that you can link up many vcs and connect them like the transition above mentioned (more advanced topic).

So how are theyÂ linked?

Check out the project file. You can see Main Interface is selecting Main. Main is your Main.storyboard in your project folder.

After app is launched, it will go into this Main Interface and find the Intial View Controller to present, and in this case, it is ViewController.swift.

Open Main.storyboard.

  • You can see the Storyboard Entry Point is pointing to a UIViewController
  • This UIViewController is your ViewController.swift. Defined in the top right Custom Class. It is casting ViewController.swift (your file) to this UIViewController. That means that this UIViewController is your ViewController.swift and visualization of it.

And everything is connected and it explains why the simulator is showing white screen because the default background is white.
